02

Files Structure

After downloading and extracting template package you will find 3 folders:

  • HTML: ready to use HTML template;
  • RamioGULP: if you want to totally customize Ramio, you can use this template;
  • Documentation: this help file.

This section describes HTML folder structure. More about RamioGULP folder structure you can find out in RamioGULP section.

Ramio is one page HTML template. So, mostly you will need one *.html file and supporting *.css, *.js, *.php files with some images. All the directories and files are well organized as it shown on the image bellow.

  • all HTML files are located in the root directory;
  • separate folders are created for CSS files, JavaScript files, fonts and images;
  • source-files folder contains all expanded CSS files, expanded JavaScript files, plugins and components files;
  • mail.php file for sending e-mails is located in the root directory;
  • .htaccess file is located in the root directory. This file contain rules for files caching on web server.

Upload to server

In order to start working with Ramio, all you have to do is copy chosen .html file along with supporting folders provided with this template on your FTP server.

CSS Files

All CSS files are placed in css folder and have to be included in the <head> tag. Ramio uses compressed CSS files and inline CSS styles to improve Google PageSpeed Insights performance.

<!-- Loader Styles Start-->
<style>
    ...
</style>
<!-- Loader Styles End-->

<!-- Bootstrap Grid System Start-->
<style>
    ...
</style>
<!-- Bootstrap Grid System End-->

<!-- First Visible Screen CSS Start-->
<style>
    ...
</style>
<!-- First Visible Screen CSS End-->

<!-- Load Fonts CSS Start-->
<script>
    loadCSS( "css/fonts.min.css?ver=1.0.0", false, "all" );
</script>
<!-- Load Fonts CSS End-->

<!-- Load Custom CSS Start-->
<script>
    loadCSS( "css/main.min.css?ver=1.0.0", false, "all" );
    loadCSS( "css/colors/color-1.min.css?ver=1.0.0", false, "all" );
</script>
<!-- Load Custom CSS End-->

You can use expanded CSS files from source-files → ramio-css-files folder for convenient customization.

CSS files that have to be included in chosen .html file:

  • fonts.min.css
  • main.min.css
  • color-*.min.css

* - color skin number

JavaScript Files

All JavaScript files are placed injs folder and have to be included before the </body> tag.

  • libs.min.js - all JavaScript libraries that are necessary for Ramio proper work, are сompiled in this file;
  • gallery-init.js - for photoswipe gallery initialization;
  • ramio-custom.js - includes other scripts calls and settings;
  • map-*.js - for Google Map.
<!-- Load Scripts Start-->
<script type="text/javascript" src="js/libs.min.js"></script>
<script type="text/javascript" src="js/gallery-init.js"></script>
<script type="text/javascript" src="js/ramio-custom.js"></script>
<script type="text/javascript" src="js/maps/map-color-1.js"></script>
<script type="text/javascript" src="https://maps.googleapis.com/maps/api/js?key=AIzaSyDNMDtmEsvSevM4ztfsbhLfLNZhKHCvWXk"></script>
<!-- Load Scripts End-->

* - color skin number

03

Fonts

In the Ramio template we used two fonts:

  • Poppins font is used for headlines and basic text content;
  • Libre Baskerville Italic is used for additional text.

Find the Load Fonts CSS section in the <head> tag and make sure that the file fonts.min.css is added:

<!-- Load Fonts CSS Start-->
<script>
    loadCSS( "css/fonts.min.css?ver=1.0.0", false, "all" );
</script>
<!-- Load Fonts CSS End-->

Fonts that are required to quickly load the first visible screen are also added using inline styles. You need to pay attention to this if you want to use other fonts:

<!-- First Visible Screen CSS Start -->
<style>
    <!-- First Visible Screen CSS Styles Here -->
</style>
<!-- First Visible Screen CSS End -->

04

Icons

Ramio uses Ionicons 2.0.1 iconic font. You can use ionicons.com to easily find the icon you want to use. Icons are already compiled in the fonts.min.css file and ready for use.

To change icons, just change the class names: ion-ionic, ion-arrow-up-a, ion-arrow-resize.

<li>
    <a href="https://www.facebook.com/" target="_blank">
        <i class="icon ion-social-facebook"></i>
    </a>
</li>

All available icons you can see on official website - ionicons.com.

05

Color Skins

Ramio is provided with 4 color skins and 16 ready-to-use demos.

Each color skin has its own CSS file. You can find all of them in the css → colors folder.

All CSS files are compressed, so if you want to make some changes in the color skin, use our Gulp project or find an expanded CSS file in the source-files → ramio-css-files → colors folder.

Please note that some colors that are required for the first visible screen are also added using inline styles:

<!-- First Visible Screen CSS Start -->
<style>
    <!-- First Visible Screen CSS Styles Here -->
</style>
<!-- First Visible Screen CSS End -->

The first visible screen expanded CSS files you can also find in the source-files → ramio-css-files folder.

To make sure that the desired color skin is applied, find the Load Custom CSS section in the <head> tag and make sure that the file color-*.min.css is added:

<!-- Load Fonts CSS Start-->
<script>
    loadCSS( "css/main.min.css?ver=1.0.0", false, "all" );
    loadCSS( "css/colors/color-1.min.css?ver=1.0.0", false, "all" );
</script>
<!-- Load Fonts CSS End-->

More beauriful color skins are coming soon in the next update.

* - color skin number

Loader

Loader will hide contents of your site until it's fully loaded. If you want to customize or remove loader, find this code in *.html file:

<div class="loader">
    <div class="loader-content">
        <div class="loader-logo slideInDown">
            <img src="img/ramio-loader-logo.svg" alt="Ramio">
        </div>
        <div class="loader-caption slideInUp">
            <span class="loading-dots">
                <span class="dot"></span>
                <span class="dot"></span>
                <span class="dot"></span>
            </span>
        </div>
    </div>
</div>

Loader styles are included in the <head> tag via inline styles. If you want to customize loader styles, you can use our Gulp project or find expanded CSS file like loader-*.css in source-files → ramio-css-files → loaders folder.

* - color scheme number.

Countdown Setup

Find the Load Scripts section at the end of <body> and make sure that all the necessary js files are added in the following order:

  • libs.min.js
  • ramio-custom.js
<!-- Load Scripts Start-->
<script type="text/javascript" src="js/libs.min.js"></script>
...
<script type="text/javascript" src="js/ramio-custom.js"></script>
...
<!-- Load Scripts End-->

Setting your own date for countdown script can be done in ramio-custom.js file in js folder:

$(function () {
    $('#countdown').countdown({until: $.countdown.UTCDate(+10, 2018, 10, 7), format: 'D'});
});

Visit KBW Countdown site for more details and settings - keith-wood.name/countdown.html

Contact Form

Ramio has integrated ajax contact form. It's very easy to set up this form to receive feedback on your e-mail. First of all you need to make sure that mail.php file is added to the root directory of your project.

Then find <form> tags with the following id:

  • id="contact-form"

in the *.html file. Each form contains 3 hidden required inputs. Replace data in the value parameter of these hidden fields:

<form class="form form-light contact-form" id="contact-form">
    <!-- Hidden Required Fields-->
    <input type="hidden" name="project_name" value="Ramio Coming Soon Template">
    <input type="hidden" name="admin_email" value="example@gmail.com"> <!-- Your E-mail Here-->
    <input type="hidden" name="form_subject" value="Contact Form Message">
    <!-- END Hidden Required Fields-->
    ...
</form>

JavaScript files that have to be included:

  • libs.min.js
  • ramio-custom.js
<!-- Load Scripts Start-->
<script type="text/javascript" src="js/libs.min.js"></script>
...
<script type="text/javascript" src="js/ramio-custom.js"></script>
...
<!-- Load Scripts End-->

Done! Now you can receive messages on your e-mail.

This active subscribe form will save e-mails to Mailchimp list:

<!-- Notify Contact Form Start-->
<form class="notify-form form form-dark">
    <input class="" type="email" placeholder="Email Adress" required>
    <button class="btn btn-l btn-dark" type="submit">
        <span class="btn-caption">Send</span>
    </button>
</form>
<!-- Notify Contact Form End-->

JavaScript files that have to be included:

  • libs.min.js
  • ramio-custom.js
<!-- Load Scripts Start-->
<script type="text/javascript" src="js/libs.min.js"></script>

<script type="text/javascript" src="js/ramio-custom.js"></script>
...
<!-- Load Scripts End-->

Follow these steps to get Mailchimp list URL:

  1. Login or create an account on mailchimp.com and go to the Lists section.
  2. Create a list.
  3. For this list, go to Signup forms section.
  4. Select Embedded forms.
  5. Copy action URL from the form.
  6. Add http:// if it is not included with Mailchimp list URL.

Open ramio-custom.js file from js folder. Just replace our Mailchimp form action URL with your Mailchimp list URL:

$('.notify-form').ajaxChimp({
    callback: mailchimpCallback,
    // Replace this Mailchimp form action URL with your Mailchimp list URL
    url: 'https://besaba.us10.list-manage.com/subscribe/post?u=e8d650c0df90e716c22ae4778&id=cda9d4d43c'
});

RamioGULP

If you want to totally customize Ramio, you can use our all-inclusive RamioGULP template. RamioGULP includes Bootstrap (grid), Gulp, Sass, Bourbon, Autoprefixer, Browsersync, Clean-CSS, CSS Beautify, Uglify, Imagemin, Bower and other packages for efficient and comfortable work.

RamioGULP requires node, npm and git.

Getting started

Install Node Modules: npm i
Run RamioGULP: gulp

Project structure

  • The source code is located in the app directory;
  • SASS files are located in the sass directory;
  • Bower packages installation path - bower_components;
  • The dist folder is a production folder. It contains ready project with optimized HTML, CSS, JS and images.

Gulp tasks

gulp: run default gulp task for web development;
build: build project to dist folder;
clearcache: clear all gulp cache.

If you have any questions about RamioGULP, you can email us on support@mixdesign.club.
